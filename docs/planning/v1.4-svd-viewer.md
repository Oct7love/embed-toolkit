# v1.4 — CMSIS-SVD 查看器（svd-viewer）planning

路由：`/tools/codegen/svd-viewer`

---

## 1. 已知事实

- 现有 `BitGrid` 组件 189 行，props: `value / width / onBitToggle / labels / colors`。**没有** field（多 bit 位域）概念。
- BitGrid 已被 `register-viewer / ieee754-parser / bit-operation-generator` 三处复用。**不能破坏现有调用**。
- 项目 zero 第三方 XML 库依赖。
- CMSIS-SVD 是 ARM 标准格式，每个 STM32 厂商都发布 .svd 文件（如 STM32F103.svd 约 1MB）。
- 浏览器原生 `DOMParser` 可解析 XML，已经够用。

## 2. 假设

- A1：用户上传或粘贴的 SVD 都是**合法 XML**，不处理 XXE 攻击（纯前端，无服务端，DOMParser 默认禁用 external entity）。
- A2：文件大小限制 **10MB**（典型 SVD 文件 0.5-3MB，10MB 是宽松上限，防止意外上传 ISO/视频）。
- A3：解析后**不持久化**到 localStorage（SVD 文件大、用户可能不希望浏览器缓存）。
- A4：UI 是经典的"左树右详"——左侧 peripheral / register 树，右侧选中后显示 BitGrid + field 列表。
- A5：搜索按 register / field name 模糊匹配（不搜 description）。

## 3. 开放问题

### Q1 — BitGrid 加 fields 还是新写 BitFieldGrid？

| 选项 | 优点 | 缺点 |
|------|------|------|
| **(a) BitGrid 加可选 `fields?` prop** | 复用率高，所有 register 显示统一样式 | 改动现有共享组件，要确保不破坏 3 处现有调用 |
| **(b) 新写 `BitFieldGrid`** | 0 风险破坏现有 | 重复代码，未来不一致 |

→ **倾向 (a)**：加成可选 prop（默认 undefined），现有调用点零修改。BitGrid 内部如检测到 `fields` 就额外渲染一行"位域条"，每条按 startBit-endBit 占位。

→ **surgical change 守则**：BitGrid 已有 colors 是 `Record<number, string>`，新 fields 是 `{startBit, endBit, name, color}[]`，两者互不冲突。请确认接受方案 (a)。

### Q2 — SVD 解析多深？

CMSIS-SVD 包含：device > peripherals > peripheral > registers > register > fields > field。

| 选项 | 解析深度 |
|------|---------|
| (a) 只到 register（每个 register 显示完整 32 位 + 字段名表） | 简单 |
| (b) 完整解析到 field 级别（每个 field 显示在 BitGrid 上） | 用户期望的"专业感" |

→ **倾向 (b)**：BitGrid 加 fields prop 的目的就是这个。请确认。

### Q3 — 树展开/折叠状态保留？

→ **方案**：纯组件 state，不存 store/URL。刷新页面回到初始（peripheral 全折叠）。请确认是否够用。

### Q4 — 上传 vs 粘贴？

候选：(a) 仅文件上传；(b) 仅粘贴 textarea；(c) 都支持。

→ **倾向 (c) 都支持**：移动端粘贴更方便，桌面用文件上传更快。两个 UI 元素并列即可。请确认。

### Q5 — 是否提供示例 SVD？

→ **倾向 yes**：内置 1-2 个迷你 SVD 片段（仅含 GPIOA 等少量 peripheral，文件 < 50KB）作为"试一下"按钮，避免用户找不到文件就走人。

## 4. 简化方案考察

### S1：能不能用 `xml2js` 等库？

→ 不行。会引入一个 ~100KB 的依赖只为了一次性解析。`DOMParser` 是浏览器原生的，零依赖。

### S2：能不能不做搜索？

→ 不行。一个 STM32 SVD 几百个 register，没搜索找不到东西。

### S3：能不能不做内置示例？

→ 能但不应该。第一次访问的用户没现成的 .svd 文件就走了，转化率会低。
