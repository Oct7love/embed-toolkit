# 产品需求文档（PRD）

## 1. 产品概述

**产品名称：** Embed Toolkit
**产品定位：** 面向嵌入式开发者的浏览器端一站式工具集合平台
**核心价值：** 将嵌入式工程师日常开发中频繁用到的计算、转换、调试、可视化工具整合在一起，替代散落各处的零碎在线工具和本地脚本

### 技术约束

- 纯前端应用，所有计算在浏览器本地完成
- 无后端服务器，不依赖任何 API
- 部署到 Vercel（免费方案），开箱即用

## 2. 目标用户

| 用户类型 | 描述 | 核心需求 |
|----------|------|----------|
| 嵌入式软件工程师 | 日常写 C/C++ 代码，调试 MCU 通信协议 | 快速计算 CRC、解析协议帧、转换进制/字节序 |
| 硬件工程师 | 设计电路、选型元器件、配置 GPIO | 电阻色环计算、分压/滤波计算、引脚分配 |
| 电子信息专业学生 | 学习嵌入式开发，准备面试 | 工具辅助学习、刷面试题、理解 RTOS 调度原理 |

### 使用场景

1. **日常开发中** — 写代码时需要快速转换进制、查 ASCII 表、计算 CRC
2. **协议调试时** — 抓到一帧数据，需要按协议模板拆解各字段含义
3. **硬件设计时** — 选电阻、算分压、配置寄存器位域
4. **学习备考时** — 刷嵌入式面试题，用可视化工具理解 RTOS 任务调度

### 3.2 分类一：数据转换工具

#### 1. 进制转换器 `P0`

**路由：** `/tools/converter/base-converter`

**需求描述：**
- 支持 hex、bin、dec、oct 四种进制之间的任意互转
- 输入任一进制数值，其余三种实时联动更新
- 支持批量转换：粘贴多行数值，一次性转换
- 支持有符号/无符号整数切换（8/16/32/64 位）
- 输入非法字符时实时提示错误

**验收标准：**
- [ ] 输入任意一个输入框，其余三个实时更新
- [ ] 批量输入 100 行数据，转换结果正确
- [ ] 有符号 -1 在 8 位显示为 0xFF / 11111111 / 255 / 377
- [ ] 非法输入（如 hex 输入 G）有明确错误提示
- [ ] 结果支持一键复制

#### 2. IEEE 754 浮点解析器 `P0`

**路由：** `/tools/converter/ieee754-parser`

**需求描述：**
- 输入 4 字节（float）或 8 字节（double）的 hex 值
- 可视化拆解显示：符号位（1 bit）、指数位（8/11 bit）、尾数位（23/52 bit）
- 每个 bit 用颜色区分，鼠标悬停显示该 bit 的含义
- 同时显示解析出的十进制浮点数值
- 支持反向操作：输入十进制浮点数，显示对应的 hex 和 bit 拆解

**验收标准：**
- [ ] 输入 `41200000` 正确解析为 `10.0`（float）
- [ ] 输入 `7F800000` 正确识别为 `+Infinity`
- [ ] 输入 `FFC00000` 正确识别为 `NaN`
- [ ] 符号位、指数位、尾数位颜色区分清晰
- [ ] 支持 float/double 切换

#### 3. 字节序转换器 `P0`

**路由：** `/tools/converter/endian-converter`

**需求描述：**
- 输入 hex 字符串，在大端（Big-Endian）和小端（Little-Endian）之间转换
- 支持 16 位（2 字节）、32 位（4 字节）、64 位（8 字节）宽度选择
- 实时显示两种字节序的排列方式（带字节编号标注）
- 支持批量转换

**验收标准：**
- [ ] 输入 `0x12345678`（BE）→ 输出 `0x78563412`（LE）
- [ ] 16/32/64 位宽度切换正确
- [ ] 输入长度不足时自动补零并提示

#### 4. 校验和计算器 `P0`

**路由：** `/tools/converter/checksum-calculator`

**需求描述：**
- 支持算法：CRC-8、CRC-16、CRC-32、XOR 校验、累加和
- CRC 支持自定义参数：多项式（Polynomial）、初始值（Init）、输入反转（RefIn）、输出反转（RefOut）、异或输出（XorOut）
- 提供常用 CRC 预设（CRC-16/MODBUS、CRC-16/CCITT、CRC-32 等）
- 输入支持 hex 字符串和 ASCII 文本两种模式
- 实时显示计算结果（hex 和 dec）

**验收标准：**
- [ ] CRC-16/MODBUS 计算 `01 03 00 00 00 0A` 结果正确
- [ ] 预设参数切换后结果实时更新
- [ ] 自定义多项式计算结果可与标准工具交叉验证
- [ ] 支持大数据量输入（>1KB hex）

#### 5. ASCII/编码对照表 `P1`

**路由：** `/tools/converter/ascii-table`

**需求描述：**
- 表格展示完整 ASCII 码表（0-127），含十进制、十六进制、八进制、字符、描述
- 控制字符（0-31、127）用特殊标记显示，附中文说明
- 点击任意单元格一键复制
- 提供搜索功能：输入字符、十进制或十六进制值快速定位
- 扩展区域：常用 Unicode 字符速查（如箭头、数学符号）

**验收标准：**
- [ ] 128 个 ASCII 字符全部正确显示
- [ ] 控制字符有可读的中文描述（如 `0x0A` → 换行 LF）
- [ ] 搜索 `65` 定位到字符 `A`
- [ ] 点击复制后剪贴板内容正确

### 3.3 分类二：协议调试工具

#### 6. 串口协议解析器 `P1`

**路由：** `/tools/protocol/serial-parser`

**需求描述：**
- 粘贴 hex 数据帧（如 `AA 55 03 01 02 03 AB`）
- 用户自定义协议模板：定义帧头、长度字段、数据区、校验字段的位置和大小
- 自动按模板拆解帧：用不同颜色高亮各字段
- 支持保存和加载自定义模板
- 校验字段支持自动验证（CRC、XOR 等）

**验收标准：**
- [ ] 自定义模板能正确定义字段偏移和长度
- [ ] 拆解结果与手动计算一致
- [ ] 模板可保存到 localStorage 并恢复
- [ ] 校验失败时有红色警告提示

#### 7. MQTT 报文解析器 `P1`

**路由：** `/tools/protocol/mqtt-parser`

**需求描述：**
- 输入 MQTT 原始字节流（hex）
- 自动识别并解析 MQTT 固定头：报文类型、DUP、QoS、RETAIN
- 解析剩余长度（支持多字节长度编码）
- 按报文类型解析可变头和有效载荷（CONNECT、PUBLISH、SUBSCRIBE 等）
- 结果以树形结构展示

**验收标准：**
- [ ] 正确解析 CONNECT 报文的 Protocol Name、Keep Alive 等字段
- [ ] 正确解析 PUBLISH 报文的 Topic 和 Payload
- [ ] 剩余长度的多字节编码解析正确
- [ ] 非法报文有错误提示

#### 8. JSON 协议构造器 `P1`

**路由：** `/tools/protocol/json-builder`

**需求描述：**
- 可视化界面构建 JSON 指令帧
- 支持添加字段：key-value 对，value 支持 string/number/boolean/array/object
- 拖拽调整字段顺序
- 实时预览生成的 JSON 字符串
- 一键复制，支持压缩和格式化两种输出
- 提供模板功能：保存常用的 JSON 指令结构

**验收标准：**
- [ ] 支持嵌套 object 和 array
- [ ] 拖拽排序功能正常
- [ ] 输出 JSON 格式合法（可通过 JSON.parse 验证）
- [ ] 模板保存和加载功能正常

#### 9. Modbus 帧生成器 `P0`

**路由：** `/tools/protocol/modbus-generator`

**需求描述：**
- 选择 Modbus 功能码（01-06、15、16）
- 输入参数：从站地址、起始地址、寄存器数量/写入值
- 自动生成 Modbus RTU 帧（含 CRC-16/MODBUS 校验）
- 自动生成 Modbus TCP 帧（含 MBAP 头）
- 显示帧结构拆解（每个字段含义标注）

**验收标准：**
- [ ] 功能码 03，从站 1，起始 0x0000，数量 10 → 帧 `01 03 00 00 00 0A C5 CD`
- [ ] RTU 和 TCP 帧同时显示
- [ ] CRC 计算结果正确
- [ ] 各字段颜色区分并有 tooltip 说明

### 3.4 分类三：芯片与硬件工具

#### 10. 寄存器位域计算器 `P0`

**路由：** `/tools/hardware/register-viewer`

**需求描述：**
- 输入 32 位寄存器值（hex 或 bin）
- 可视化显示 32 个 bit 的网格，每个 bit 可点击翻转
- 支持自定义位域模板：命名每个位域的范围和含义
- 修改任意 bit 后，hex/bin/dec 值实时联动更新
- 支持保存自定义位域模板（如 STM32 GPIO_MODER 寄存器）

**验收标准：**
- [ ] 32 个 bit 网格显示正确，bit31 在最左
- [ ] 点击 bit 翻转后数值实时更新
- [ ] 位域模板能正确高亮对应的 bit 范围
- [ ] 位域模板可保存和加载

#### 11. GPIO 引脚分配表 `P2`

**路由：** `/tools/hardware/gpio-planner`

**需求描述：**
- 提供常见 MCU 型号的引脚定义（STM32F103C8T6、ESP32 等）
- 表格展示所有引脚，每行显示：引脚号、默认功能、可复用功能列表
- 拖拽或下拉选择为每个引脚分配功能
- 自动检测功能冲突（如两个引脚分配了同一个 UART TX）
- 导出 C 语言 GPIO 初始化代码

**验收标准：**
- [ ] 至少支持 STM32F103C8T6 和 ESP32 两款芯片
- [ ] 引脚复用冲突时有红色警告
- [ ] 导出的 C 代码可直接编译
- [ ] 支持保存和加载配置方案

#### 12. 电阻色环计算器 `P1`

**路由：** `/tools/hardware/resistor-calculator`

**需求描述：**
- 正向计算：选择色环颜色（4/5/6 环）→ 计算阻值和精度
- 反向查询：输入阻值 → 显示对应的色环颜色组合
- 色环选择器使用实际颜色渲染（黑棕红橙黄绿蓝紫灰白金银）
- 显示 E 系列标准阻值中最接近的值
- 可视化显示电阻外观（带色环的电阻图）

**验收标准：**
- [ ] 4 环电阻：棕黑红金 → 1kΩ ±5%
- [ ] 反查 4.7kΩ → 显示黄紫红
- [ ] 色环颜色渲染准确
- [ ] E24 系列推荐值正确

#### 13. 分压/RC 滤波计算器 `P1`

**路由：** `/tools/hardware/rc-calculator`

**需求描述：**
- **分压计算器：** 输入 Vin、R1、R2 → 计算 Vout，或输入目标 Vout 反推 R2
- **RC 低通滤波器：** 输入 R、C 值 → 计算截止频率 fc
- **RC 高通滤波器：** 输入 R、C 值 → 计算截止频率 fc
- 绘制波特图（幅频特性），使用 Recharts
- 电阻支持 E 系列标准值选择，电容支持常见值选择

**验收标准：**
- [ ] 分压：Vin=5V，R1=10kΩ，R2=10kΩ → Vout=2.5V
- [ ] RC 低通：R=10kΩ，C=10nF → fc≈1591.5Hz
- [ ] 波特图幅频曲线形状正确（-20dB/dec 斜率）
- [ ] 反推电阻值计算正确

### 3.5 分类四：RTOS 可视化工具

#### 14. 任务调度甘特图 `P2`

**路由：** `/tools/rtos/task-scheduler`

**需求描述：**
- 输入任务列表：任务名、优先级、周期（ms）、执行时间（ms）
- 模拟 FreeRTOS 抢占式优先级调度算法
- 生成时序甘特图：X 轴为时间，Y 轴为任务，用色块表示运行/就绪/阻塞状态
- 支持调节仿真时间范围
- 检测并提示 CPU 利用率过高或任务错过 deadline 的情况

**验收标准：**
- [ ] 3 个不同优先级的周期任务调度结果正确
- [ ] 高优先级任务能正确抢占低优先级任务
- [ ] 甘特图色块对齐、可缩放
- [ ] CPU 利用率计算正确

#### 15. 内存布局可视化 `P2`

**路由：** `/tools/rtos/memory-layout`

**需求描述：**
- 手动输入模式：添加内存段（段名、起始地址、大小、类型）
- 文件导入模式：粘贴 GCC .map 文件内容，自动解析 `.text`、`.data`、`.bss`、`.heap`、`.stack` 等段
- 可视化显示内存布局：堆叠的彩色色块，标注段名、地址范围、占用比例
- 分别展示 Flash 和 RAM 的使用情况
- 显示总占用率和剩余空间

**验收标准：**
- [ ] 手动添加段后布局图正确显示
- [ ] 能解析标准 GCC arm-none-eabi .map 文件
- [ ] Flash/RAM 占用率百分比正确
- [ ] 段地址有重叠时有警告提示

### 3.6 分类五：代码辅助工具

#### 16. 位操作代码生成器 `P1`

**路由：** `/tools/codegen/bit-operation`

**需求描述：**
- 可视化 32 位网格，勾选要操作的 bit 位

- 支持指定寄存器名称（如 `GPIOA->ODR`）

**验收标准：**
- [ ] 勾选 bit3 和 bit7，SET 操作 → `REG |= (1 << 3) | (1 << 7)`
- [ ] 清零操作使用 `&= ~(...)` 语法
- [ ] 生成的宏定义格式规范
- [ ] 自定义寄存器名正确替换到代码中

#### 17. 状态机编辑器 `P2`

**路由：** `/tools/codegen/state-machine`

**需求描述：**
- 可视化画布：拖拽添加状态节点、拖拽连线表示状态转移
- 每个转移线可标注触发事件和动作
- 支持设置初始状态和终止状态
- 自动生成 C 语言代码：基于 switch-case 的状态机框架
- 支持导出状态转移表

**验收标准：**
- [ ] 拖拽创建状态和转移线操作流畅
- [ ] 生成的 C 代码结构完整（enum 定义、switch-case、事件处理函数）
- [ ] 状态转移表与可视化图一致
- [ ] 支持保存和加载状态机方案
